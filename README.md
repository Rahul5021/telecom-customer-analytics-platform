# Telecom Customer Analytics Platform

## Project Status

Project initialized.  
Dataset collected.  
Environment configured.  

Development in progress.

---

## 1. Problem Statement

This project simulates a subscription-based telecom company experiencing rising customer churn.

The goal is to understand:

- Why customers leave  
- Which customers are likely to churn  
- How much revenue is at risk  
- What actions can reduce churn  

---

## 2. Objective

Build an end-to-end analytics and machine learning platform that:

- Cleans and prepares telecom customer data  
- Performs structured SQL-based churn analysis  
- Conducts revenue and retention analytics  
- Builds a predictive churn model  
- Provides business-focused insights  

---

## 3. Dataset

**Source:** Kaggle Telecom Churn Dataset  

The dataset contains customer-level data including:

- Demographics  
- Contract information  
- Billing details  
- Service usage  
- Churn status (Target Variable)  

**Target Variable:**  
`Churn` — Binary classification (Yes/No)

---

## 4. Planned Architecture

The project will include:

1. Data Cleaning & Transformation  
2. Relational Modeling  
3. SQL-based Churn & Revenue Analysis  
4. Machine Learning Pipeline  
5. Model Evaluation & Explainability  
6. Business Impact Simulation
